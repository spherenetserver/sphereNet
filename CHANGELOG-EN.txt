[2026-05-31] - NPC AI tick-cost bounding under crowded combat (Wave 30)
----------------------------------------

PERFORMANCE:
  - Per-tick NPC budget: a mass sector activation (a player entering a densely
    populated area) could make thousands of NPCs due in one tick and run their
    AI serially, stalling the tick for several seconds. The tick now processes
    at most a bounded number of NPCs and defers the overflow to the next tick.
  - Bounded target acquisition: FindBestTarget previously did a line-of-sight
    raycast (against the real static map — the dominant cost) for every nearby
    mobile, making acquisition O(crowd) per NPC. It now ranks candidates by
    motivation without LOS, caps how many are evaluated, and verifies LOS lazily
    only on the top few — a few raycasts per NPC regardless of crowd size.
  - Net: a pathological stress (50,000 monsters clustered around 8 fighting
    players) dropped from ~8.8s ticks to ~2.2s; normal and realistic loads are
    unaffected. (525 tests pass.)

[2026-05-31] - Headless 'stress' console command (Wave 29)
----------------------------------------

ADMIN / DIAGNOSTICS:
  - Added a `stress [items] [npcs]` console command mirroring the in-game
    .STRESS GM speech command, so the load-test population can be generated
    from a headless server (CI / smoke). Routes to StressTestEngine.QueueGenerate.
  - Smoke-verified on the real script/map data: generating 8,000 NPCs took ~1s
    and the tick loop stayed healthy (avg ~0.6ms, 8,006 chars + 6 players) with
    zero errors. (Note: stress NPCs are neutral load dummies, so combat bots —
    which only engage hostile monsters — do not fight them.)

[2026-05-31] - Headless 'bot' console command (Wave 28)
----------------------------------------

ADMIN / DIAGNOSTICS:
  - Added a `bot <count> [walk|combat|idle|smart]` console command so the bot
    stress/smoke framework can be driven from a headless server (stdin / CI),
    not only via the in-game .BOT GM speech command. Routes to the same spawn
    path. Verified end-to-end: bots auto-create accounts, create characters,
    enter the world, and walk — with the movement validator correctly rejecting
    water/impassable steps and fast-walk against the real map data.

[2026-05-31] - .NET 10 migration, ServerGC, deterministic test suite (Wave 27)
----------------------------------------

RUNTIME / BUILD:
  - All SphereNet projects retargeted net9.0 -> net10.0 (LTS). Full solution
    builds clean; 525 tests pass under net10.
  - ServerGarbageCollection + ConcurrentGarbageCollection enabled on the Host
    entry point (the game Server already had them).

TESTS (flaky CI fix, no tests removed):
  - Root-caused the intermittent CI failures to process-wide mutable static
    hooks left set across tests (ObjBase/Item.ResolveWorld,
    Character.ResolveAccountForChar which drives PrivLevel, VendorEngine.World).
    Under xUnit's non-deterministic order a later test resolved against a stale
    hook (e.g. a player inheriting another test's GM account).
  - Added a BeforeAfterTest attribute that resets those hooks before/after every
    test, plus assembly-wide serialization. The suite is now green across
    repeated runs regardless of order.

[2026-05-31] - D-prefix fix extended to function calls (Wave 26)
----------------------------------------

SCRIPT ENGINE:
  - The D-prefix fix (Wave 24) now also covers D-prefixed FUNCTION calls: a
    [FUNCTION] whose name starts with 'd' (e.g. DamTypesOfString) was losing its
    leading 'd' and never resolving. The D-handler fallback now resolves the
    full token via the function resolver as well as the variable resolver.

TESTS:
  - ExpressionParser regression extended with a D-prefixed function call.

[2026-05-31] - No-space <Rnnn> random expression form (Wave 25)
----------------------------------------

SCRIPT ENGINE:
  - <Rnnn> (no space, e.g. <R999>) is now accepted as the random 0..nnn-1 form,
    matching the spaced <R nnn>. 'R' followed by a digit is random; 'R' followed
    by a letter (REF, REGION, ...) is left alone. Previously the no-space form
    was unresolved and read 0, so IF !(<R999>) was always true (the rare branch
    fired every time).

TESTS:
  - ExpressionParser regression test for the no-space <R5> form (spaced <R 5>
    still works).

[2026-05-31] - Bare <dispid>/<dex> expression resolution fix (Wave 24)
----------------------------------------

SCRIPT ENGINE:
  - The "D force-decimal" prefix was wrongly swallowing bare self-properties
    that start with 'd' (DISPID, DEX, DIR, ...): it stripped the leading 'd'
    and resolved the remainder (<dispid> -> <ispid> -> 0). It now tries the
    'd'-stripped remainder first (preserving the legitimate <DHITS>/<DLOCAL.HEX>
    decimal form) and, only if that is unresolved, resolves the full token as a
    property.
  - Added an Item <DISPID>/<ID> read property (reflects the DISPID override,
    defname-or-hex form like BASEID). There was no read before, so
    IF (<dispid> == i_xxx) weapon-type bonus dispatch and furniture open/close
    (DISPID toggle) did not work.

TESTS:
  - ExpressionParser regression test that <dispid>/<DEX> are not mangled while
    the <DHITS> decimal form is preserved (524 passing).

[2026-05-31] - Script intrinsics RESOURCETYPE / RESOURCEINDEX (Wave 23)
----------------------------------------

SCRIPT:
  - <RESOURCETYPE defname> and <RESOURCEINDEX defname> resolve a defname to its
    resource type code and index. The type uses the Source-X RES_* numbering
    (CharDef=6, ItemDef=14, Spawn=37, ...) so that
    <RESOURCETYPE <entry>> == <def.res_chardef> compares correctly. The worldgen
    spawner's NPC/item discrimination and staff restype menus rely on this.
    Resolved via ResolveDefName; an unknown defname returns 0 (RES_UNKNOWN).

[2026-05-31] - Region periodic triggers @RegPeriodic / @CliPeriodic (Wave 22)
----------------------------------------

REGION TRIGGERS:
  - @CliPeriodic fires periodically for every online player on their current
    region (SRC = the player). @RegPeriodic fires once per tick for each region
    holding at least one player (SRC = a representative player), mirroring
    Source-X CSector's iRegionPeriodic environ tick; uninhabited regions never
    tick. Region ambient sound/music scripts (ON=@RegPeriodic SRC.SOUND,
    ON=@CliPeriodic SRC.MUSIC) now run. Reuses the existing FireRegionEvents
    mechanism (same path as Enter/Leave/Step).

[2026-05-31] - NPC speech broadcast (SAY/SAYU/EMOTE) (Wave 21)
----------------------------------------

NPC SPEECH:
  - SAY / SAYU / SAYUC / EMOTE now BROADCAST as overhead speech to nearby
    clients instead of a private system message to the source (SAY = ASCII
    0x1C, SAYU/SAYUC = Unicode 0xAE, EMOTE = emote type). An optional
    @color,font,mode prefix is parsed for the hue. NPC dialog/bark lines were
    previously invisible because they only went to the source as a system
    message.

[2026-05-31] - NPC AI morale/kiting and vendor restock unification (Wave 20)
----------------------------------------

NPC AI:
  - Morale/fear (Source-X NPC_GetAttackContinueMotivation): the decision to
    flee is now based on RELATIVE strength instead of a fixed "HP < half"
    threshold — STR difference + relative HP% difference. A strong, healthy
    NPC presses a weak target even below half HP; a weak/hurt NPC flees an
    overwhelming one. Applied as a penalty only.
  - Ranged kiting (Source-X NPC_FightArchery): when a target closes inside a
    ranged NPC's minimum range, the NPC backs off (~50%) to reopen the gap
    instead of standing locked in melee unable to fire.
  - NpcAIFlags bit values aligned to the NPC_AI_* defines (0x20 gap kept), so
    a raw script NPC.AI integer maps to the correct bits in future.

VENDOR:
  - GM VENDORINV restock now writes to the virtual stock container (LAYER 26)
    that the buy gump reads from, not the regular backpack — so GM-defined
    stock both displays and stays excluded from the world save.
  - Restocked items stamp PRICE from the itemdef VALUE; the two restock paths
    (template SELL= and GM VENDORINV) now share the same virtual stock model.

TESTS:
  - Unit test for the morale penalty (weak flees / strong presses); vendor
    trade tests moved to a serial collection to avoid a shared-static race
    (523 passing).

[2026-05-29] - Vendor virtual stock and lazy NPC loot materialisation (Wave 19)
----------------------------------------

VENDOR STOCK (VIRTUAL):
  - The vendor stock container (LAYER 26/27) and its items are no longer
    written to the world save; they are rebuilt from the SELL template
    (VENDOR_SELL_LIST tag) when a player opens the vendor. This removes
    thousands of transient stock records from saves with many vendors.
  - Buying decrements the stock amount; a depleted entry is removed, and the
    container is repopulated from the template on the next open once empty.
  - Buy now validates the stock serial against the vendor's own stock
    container, preventing a crafted packet from "buying" an arbitrary world
    item by serial.
  - Stock price is stamped from the itemdef VALUE; an item with no PRICE tag
    now uses the same fallback price as the display instead of 0 (the server
    side previously rejected such items and broke the purchase).

NPC LOOT (LAZY):
  - NPC plain loot (non-wearable, non-ITEMNEWBIE chardef ITEM= entries) is no
    longer materialised onto the living NPC at spawn; it is rolled straight
    into the corpse at death (Source-X CTRIG_CreateLoot parity). Living NPCs
    carry no transient loot items in the world save.
  - Wearable gear (weapons/armour) is still equipped at spawn (combat /
    appearance) and reaches the corpse via the normal unequip path.

TESTS:
  - Regression tests for vendor virtual stock: decrement, depletion removal,
    non-stock serial rejection, over-stock amount rejection, price fallback
    (522 passing).

[2026-05-29] - NPC AI: Source-X parity fixes and smarter behaviors (Wave 18)
----------------------------------------

NPC AI - CORRECTNESS / PARITY:
  - Flee logic: a fearful NPC (negative motivation) now routes to ActFlee
    instead of silently dropping its target.
  - Multicore ApplyDecision is sorted by NpcUid before applying, removing the
    non-deterministic completion-order apply and matching single-thread behavior.
  - _losFailCounts leak: separate purge pass for NPCs that never built a path
    cache entry.
  - Multicore reschedule now checks IsDead (symmetry with single-thread), so a
    dead NPC is not re-queued into the timer wheel.
  - Self-kill (reactive-armor reflect): killer==victim is attributed as null
    (no Kill trigger on self).
  - Beneficial-spell targeting fix: ChooseBestSpell now returns the cast target,
    so heal/cure go to the right recipient. Previously every spell was cast on
    the enemy, so an NPC "self-heal" actually healed its enemy.

NPC AI - SMARTER BEHAVIORS:
  - Spell combo chain: Paralyze -> Explosion -> EnergyBolt -> Poison while the
    target is locked down (COMBO_STEP/COMBO_TARGET state).
  - Ally support: caster NPCs heal/cure a wounded nearby ally, not just self.
  - Hidden-target pursuit: when a target hides, the NPC moves to its last known
    spot and tries Reveal instead of giving up immediately.
  - ReacquireDelay: target-less NPCs throttle the full-range scan (reset on being
    attacked, so retaliation is never delayed).
  - Pet mode revert: after an attack target dies, the pet returns to its previous
    order (Guard/Follow) instead of trailing the master.
  - Threat targeting (Source-X NPC_AI_THREAT): NPCs prefer whoever has dealt
    them the most damage, driven by the attacker list (the previously-unused
    NpcAIFlags.Threat is now honored).
  - FightMode bias (FIGHTMODE tag): weakest / strongest / evil target preference.
  - Dynamic tick: badly-hurt NPCs act slightly slower.
  - Backoff: timid animals back off from the nearest threat to a safe distance.
  - DetectHidden: guards / high-INT NPCs periodically reveal hidden players.
  - Stuck -> Teleport: a stuck caster blinks toward the target instead of giving
    up.
  - Forced target: bard Provocation (PROVOKED_TARGET) and scripted ConstantFocus
    (CONSTANT_FOCUS) override normal target selection.
  - Scout retreat: creatures with Hiding vanish mid-flee to break pursuit.
  - Looting (Source-X NPC_AI_LOOTING): looter NPCs scavenge items from nearby
    corpses into their pack (LOOTING tag or global flag).
  - IntFood (Source-X NPC_Act_Food): hungry NPCs eat food from their pack or
    graze (INTFOOD tag or global flag).
  - Spellbook spell loading (Source-X NPC_AddSpellsFromBook): a caster NPC with
    no scripted spell list derives one from a carried/equipped magery spellbook.
    (Wand-holding NPCs already cast reagent-free via the wand path.)
  - Hireling wage (Source-X NPC_OnHirePay): a hired NPC (HIRE_WAGE tag) is paid
    from the master's bank box on a timer and deserts when it can't be paid;
    hirelings are exempt from hunger-based loyalty loss.

DOCS:
  - docs/ restructured: added ARCHITECTURE, STAFF_COMMANDS, TRIGGERS and a docs
    index; ADMIN_SECURITY folded into DEPLOY; stale internal notes removed.
  - README rewritten in English with README-TR.md (Turkish).

[2026-05-29] - Gameplay bug fixes and Source-X parity completion (Wave 17)
----------------------------------------

SCRIPT ENGINE:
  - Numeric brace-range {n m}: random-range and weighted {a w b w} selection in
    the expression parser; CharDef stat lines now roll between min/max.
  - FOR argument forms: FOR x (1..x), FOR min max, FOR var max, FOR var min max,
    with inclusive ranges, count-down, and a named loop variable.
  - MAX/MIN intrinsics, the @ power operator, and the numeric 3-way
    QVAL(v1,v2,lt,eq,gt) form.
  - ItemDef TDATA1/TDATA3 defname capture with lazy resolution.

COMBAT:
  - Era-0 hit-chance now factors in target DEX and a ranged-weapon adjustment.
  - Era-aware damage bonus is percentage-based; tactics/anatomy excluded in era-0.
  - Damage min>max guard prevents a Random.Next exception.
  - Weapon parry with a shield or a one/two-handed weapon.
  - Reactive-armor reflected damage is now attributed to the attacker.

MAGIC:
  - Spell-success scale fix: SKILLREQ is checked at the correct scale (high-skill
    casters no longer fizzle constantly).
  - Implemented CreateFood and DispelField; MindBlast now uses the INT-difference
    formula; EvalInt scales spell damage.
  - Cast-recovery cooldown plus a light skill-based faster-casting effect.
  - Field spells form a multi-tile wall and deal periodic tick damage to
    stationary targets.
  - Incognito disguises the real name and restores it on expiry.
  - Recall/Gate are blocked while frozen/jailed and against blocked destination
    tiles.
  - Magic-resist circle aligned with GetCircle().

SKILLS:
  - GMs no longer accrue skill/stat gains.
  - Taming difficulty rescaled to 0-100, so high-HP creatures are tameable.
  - Stealing fails without a backpack, preventing item loss and a free criminal
    flag.
  - Crafting: exceptional durability bonus, resource hue transfer, and randomized
    failure resource loss.
  - RemoveTrap springs the trap on failure; bandages gain an anatomy bonus and
    produce bloody bandages; Veterinary routes through Healing.
  - SkillEngine uses Random.Shared for multicore thread safety.
  - Gathering no longer yields more than a depleted node holds.

TRIGGERS:
  - @ReceiveItem / @NPCAcceptItem fire when giving an item to an NPC.
  - @Criminal (cancellable), @SeeCrime (nearby NPCs), @SpellInterrupt, @Hunger.

ITEMS:
  - Item CONT= / Z= / LAYER= setters for script-driven container moves.
  - DUPE verb; character CURE / REVEAL / ATTACK verbs; character ARMOR/AR read.
  - Crops/foliage harvesting, seed planting, and empty-pitcher filling from water.
  - Light sources: in-container ignite block and charge consumption; switch
    graphic swap; food/drink single-unit consumption.

WORLD:
  - Timed jail auto-release (DateTime-based, reboot-safe) plus expired-jail
    cleanup on login.
  - Party: proper disband notification and honoured invite-accept result.
    Guild: leave privilege check, candidate->master promotion, multi-guild block.
  - Housing: transfer cap, footprint passability, and IDOC lockdown/secure item
    preservation.
  - Zero-value creatures grant no fame/gold, preventing farming.

CHARACTER:
  - Poison ticks broadcast health/damage packets and attribute the kill.
  - HP regen halts while poisoned; starvation drains stamina/hits; FOOD is
    clamped; resurrect leaves at least 1 HP; pet control-slot summation and
    loyalty warnings.
  - Movement shove now costs stamina exactly once (fixing the double-deduction /
    false-block bug).

TESTS:
  - Regression tests for brace-range, MAX/MIN/@, and QVAL (515 passing).
